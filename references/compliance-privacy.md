# 合规与隐私 — 产品设计默认内建

> 涉及用户数据的功能，合规不是上线前补材料，是需求阶段就要标注的风险。

## 1. 适用法规要点（中国个保法 / GDPR 通用）

| 原则 | 要求 | 产品动作 |
|------|------|----------|
| 合法正当必要 | 只收集功能必需的数据 | 数据最小化，不「先收了再说」 |
| 告知同意 | 收集前明确告知用途并获同意 | 隐私弹窗、权限申请说明 |
| 目的限制 | 不超告知范围使用 | 数据用途变更需重新授权 |
| 数据主体权利 | 可查询/更正/删除/导出 | 提供自助入口 |
| 安全存储传输 | 加密、最小权限 | 脱敏、HTTPS、访问控制 |
| 跨境传输 | 特殊限制 | 评估是否触发，必要时单独方案 |

## 2. 产品设计合规检查清单（门禁）

- □ 这个功能收集了哪些用户数据？是否「最小必要」？
- □ 收集前是否有明确告知 + 主动同意（非默认勾选）？
- □ 数据用途是否超出告知范围？变更是否重新授权？
- □ 用户能否查询/更正/删除/导出自己的数据？
- □ 存储与传输是否加密、是否脱敏、访问是否最小权限？
- □ 是否涉及敏感个人信息（人脸/位置/健康/未成年人）？需更强同意。
- □ 是否涉及跨境？是否触发额外审批？
- □ 数据留存期限是否设定？到期是否自动删除？
- □ 日志/埋点是否含可直接识别个人的字段（如明文手机号）？

## 3. PRD 中如何标注

在 PRD 第⑧章「风险与依赖」设「合规风险」子类，并在涉及数据的需求后加标注，如：
`[合规] 搜索日志含用户 ID 与查询词，需脱敏存储，留存 ≤30 天，已确认法务。`

## 4. AI 功能额外注意

- 生成内容版权与偏见风险（见 `references/ai-feature-notes.md`）。
- 训练/调用是否含用户个人数据，是否满足「可删除」要求。
- 向用户披露「由 AI 生成」，避免误导。

## 5. 红线

未经同意收集敏感个人信息、默认勾选、数据超范围用于训练或出售 —— 任一项都是高危合规事故，必须在 PRD 阶段拦截。
