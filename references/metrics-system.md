# 指标体系 — 北极星 / 驱动 / 健康三层架构

> 没有可衡量就没有可管理。指标必须可观测、可归因、有基线和目标值。

## 1. 三层指标体系

| 层 | 作用 | 特征 | 示例（协作工具） |
|----|------|------|------------------|
| **北极星指标** | 衡量产品长期价值的唯一核心 | 1 个，代表用户真实获得的价值 | 周活跃协作团队数 |
| **驱动指标** | 拆解北极星，指导日常动作 | 3-7 个，可干预 | 首次协作完成率、文档创建数、搜索使用率 |
| **健康指标** | 防止「为增长损害体验」 | 护栏，劣化要报警 | 7 日留存、Crash 率、NPS、加载 P99 |

**关键纪律**：只盯北极星易作弊（如弹窗强推），必须用健康指标做护栏。驱动指标上升但健康指标下降 = 虚假繁荣。

## 2. 指标定义模板（每个指标必填，见 assets/metrics-template.md）

```
指标名：
定义：用户视角的精确描述
公式：分子/分母、口径（去重？窗口？）
基线值：当前实测
目标值：上线后目标 + 时间窗
数据源：埋点/表/看板
负责人：谁维护
更新频率：日报/周报
```

**完整性检查（门禁）**：缺基线、缺目标、缺数据源、缺负责人 → 不通过。

## 3. 选取原则

- 北极星 = 「用户用产品完成了什么有价值的事」，不是「我们卖了多少」。
- 领先指标（过程，当天可观测）与滞后指标（结果，1-4 周）配对使用。
- 指标数克制：北极星 1 + 驱动 ≤7 + 健康 ≤5，过多等于没有重点。
- 反例：「提升用户体验」——不可衡量；「增加功能数」——虚荣指标。

## 4. 与 PRD 的衔接

PRD 第⑦章「成功指标」必须引用本体系：每条 P0 需求至少对应 1 个驱动/健康指标，给出基线→目标。上线后第 14 天复盘即用此处定义的目标值判定达成与否。
