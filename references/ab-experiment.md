# A/B 实验设计 — 用数据证伪假设

> 验证阶段信数据。A/B 是 PM 最该掌握的「证伪」工具：不是证明你对，是看用户用脚投票。

## 1. 何时做实验

- 两个方案各有权衡、无法靠直觉判断时。
- 改动可能影响核心指标（转化/留存/收入）时。
- **何时不做**：纯体验微调（用灰度+定性）、基础设施改动（用监控）、样本太小跑不出显著结果。

## 2. 实验设计四要素

| 要素 | 说明 | 易错点 |
|------|------|--------|
| 单一变量 | 实验组与对照组只差一个变量 | 同时改 UI+算法，无法归因 |
| 随机单元 | 按用户/设备随机，非按会话 | 同一用户跨组导致污染 |
| 样本量 | 基于 MDE 和现有流量反推 | 流量小却想测 1% 差异 → 跑半年 |
| 显著性 | 设定 α=0.05，p<0.05 才结论 | 看到「趋势变好」就停，忽略置信区间 |

**MDE（最小可检测效应）**：先定「多大的提升才有业务意义」，再反推需要多少样本/天数。不要先有数据再倒推「显著」。

## 3. 假设定义模板

```
原假设 H0：方案 B 与 A 在[指标]上无差异
备择 H1：B 优于 A
主指标：[指标名]  MDE：≥[X%]
护栏指标：[健康指标] 不可显著劣化
样本：每组 ≥[N] 用户  周期：≥[D] 天（覆盖完整周循环）
决策规则：p<0.05 且主指标+，护栏不劣 → 全量；否则回滚/迭代
```

## 4. 结果解读

- **显著且正向** → 全量，记入复盘。
- **不显著** → 样本不足则延长；充足则「无差异」，别强行解读为「略好」。
- **显著但负向** → 回滚，这是实验的价值（避免了全量踩坑）。
- **主指标+但护栏-** → 不发布，先解决护栏问题。
- 警惕辛普森悖论（分组看都好、汇总变差）和周末效应（实验期需覆盖完整周）。
