# 质量门禁清单 — PRD 自检 + 评审准备

> 门禁不通过不得交付。缺项必须回补或显式标注「待确认/未提供」。

## PRD 质量自检清单（阶段三门禁）
- [ ] 问题陈述：陌生人 30 秒能看懂？
- [ ] 目标：每个都 SMART 可衡量？
- [ ] Non-Goals：≥3 条且每条有「为什么不做」？
- [ ] 用户故事：角色-行为-价值三者齐全？
- [ ] 验收标准：每个 P0 需求有 Given-When-Then？
- [ ] 异常状态：空/加载/错误/边界/权限 全覆盖？
- [ ] 指标：每条有基线 + 目标值 + 时间窗？
- [ ] 隐私合规：涉及用户数据处已标注（见 compliance-privacy）？
- [ ] 技术可行性：Tech Lead 已初步确认？
- [ ] 外部依赖：已标注且对方已知晓？

## 评审准备清单（阶段四门禁）
- [ ] 技术预评审已通过（TL 确认可行/更新 Effort）？
- [ ] 设计方向已对齐（如有设计需求）？
- [ ] PRD 自检清单全过？
- [ ] 待决策问题已列决策人 + 截止时间？
- [ ] 开发估算已确认（X 人天，X 号到 X 号）？
- [ ] 会议议程与参会人已发出？

## 上线验收清单（阶段六门禁，摘引）
- [ ] P0 用户故事逐条验收通过
- [ ] 五类异常场景通过
- [ ] 埋点正确上报、参数完整
- [ ] 灰度自动停止条件已设定
- [ ] 指标看板已配、复盘日历已建（第 7/14 天）

## 指标定义完整性检查（metrics-system 门禁）
- [ ] 每个指标有：定义 / 公式 / 基线 / 目标 / 数据源 / 负责人
- [ ] 北极星 ≤1、驱动 ≤7、健康 ≤5
- [ ] 护栏指标（健康）已设，防虚假增长
